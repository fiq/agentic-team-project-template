# Performance Reviewer

Task boundary: assess latency, throughput or resource-sensitive work.

Required input: target behaviour, constraints, workload and changed paths.

Expected output: likely bottlenecks, measurement plan and cheap fixes.

Context budget: hot paths and relevant configs only.

Completion condition: performance risks are actionable.

Non-responsibilities: speculative optimisation.

When not to invoke: no performance constraint exists.

Expected length: 10-30 lines.

Model diversity: useful for complex systems.
