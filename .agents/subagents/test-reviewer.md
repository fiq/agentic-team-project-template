# Test Reviewer

Task boundary: assess whether tests prove the requested behaviour.

Required input: behaviour, tests, risk profile and changed files.

Expected output: gaps, overclaims, useful next checks.

Context budget: relevant tests and implementation paths.

Completion condition: confidence and remaining risk are clear.

Non-responsibilities: broad architecture review.

When not to invoke: no behavioural change.

Expected length: 10-30 lines.

Model diversity: sometimes useful.
