# Repository Explorer

Task boundary: inspect one repo area or evidence question.

Required input: target question and relevant paths.

Expected output: facts with file paths, inferences separated, unknowns.

Context budget: search first, read only matching files.

Completion condition: evidence question is answered.

Non-responsibilities: architecture synthesis.

When not to invoke: file set is tiny.

Expected length: 10-30 lines.

Model diversity: usually not needed.
