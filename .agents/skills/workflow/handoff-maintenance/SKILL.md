# Handoff Maintenance

## Purpose

Keep `HANDOFF.toon` useful for session continuity without turning it into a
wiki dump.

## Include

- current objective
- current phase
- completed work
- next actions
- active assumptions
- blocking questions
- known risks
- files changed
- tests run
- knowledge consulted by ID
- knowledge proposals created by path or ID
- handoff warnings

## Exclude

- full knowledge entries
- chronological logs
- copied wiki pages
- duplicate policy
- trivial implementation notes

## When To Update

- before significant work when current state is stale
- after material decisions
- after important test or implementation results
- before ending a session

Reference knowledge IDs instead of copying entries.
